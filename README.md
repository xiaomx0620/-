# 日语视频自动字幕生成工具

选一个日语视频，自动生成带中文翻译的SRT字幕文件，还能提前告诉它人名和术语怎么翻。


## 这个工具是干什么的？

你给它一个日语视频，它还你一个带时间轴的中文字幕文件（.srt）。

整个过程全自动：
1. 先用语音识别把视频里的日语对话转成文字
2. 然后调用多个翻译引擎把日语翻成中文
3. 最后生成一条一条带时间轴的字幕文件


## 它跟别的字幕工具比有什么不一样？

最大的特点：你可以提前告诉它人名和术语怎么翻。

比如你正在做《间谍过家家》的字幕，你可以在开始前告诉工具：

ロイド・フォージャー -> 劳埃德·福杰（黄昏）
ヨル・フォージャー -> 约尔·福杰（荆棘公主）
アーニャ -> 阿尼亚
西国 -> 西国
おはよう -> 早安

工具在翻译时会先在本地把这些词强制替换掉，再送去翻译引擎。不管是有道、百度还是SiliconFlow，都会收到已经替换好的文本，确保人名术语统一。


## 主要功能模块

### 阶段一：语音识别
- 默认使用 SenseVoiceSmall 云端模型（SiliconFlow 平台），支持日语
- 也可以切换到 TeleAI/TeleSpeechASR 云端模型（日语识别更准确）
- 支持 Whisper 本地识别（日语识别成熟稳定）
- 支持主备切换

### 阶段二：本地术语替换（核心特色）
- 自动解析你填写的术语表（日文 -> 中文）
- 逐句匹配，长词优先替换
- 支持整句命中：直接使用预设译文，跳过翻译环节
- 使用占位符技术，防止替换内容被翻译引擎二次改写

### 阶段三：多引擎翻译 + 智能择优
- 同时调用多个翻译引擎：有道智云、百度翻译、DeepLX、SiliconFlow
- 每个引擎的结果会通过质量评分 + 相似度比较智能选择最优译文

### 阶段四：生成SRT字幕
- 自动生成标准格式的 .srt 字幕文件
- 保存在视频同目录下


## 安装与配置

### 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / Windows 11（64位） |
| Python 版本 | 3.9 或更高版本 |
| 内存 | 4 GB 以上（推荐 8 GB） |
| 硬盘空间 | 5 GB 以上可用空间 |
| 网络 | 需要联网（调用云端 API） |
| GPU | 不需要（使用云端识别，普通电脑即可） |

### 安装步骤

1. 安装 Python 3.9+（安装时勾选 Add Python to PATH）

2. 下载本脚本 subtitle_tool.py

3. 打开终端（命令提示符），进入脚本所在目录

4. 运行 python subtitle_tool.py

5. 点击界面上的 部署环境 按钮，自动安装所有依赖包

6. 点击 设置，填写各翻译服务商的 API 密钥

### 依赖包列表

| 包名 | 作用 |
|------|------|
| faster-whisper | Whisper 本地语音识别 |
| pydub | 音频分割和处理 |
| av | 读取视频文件中的音频流 |
| opencc-python-reimplemented | 繁体转简体 |
| requests | 调用云端 API |
| tqdm | 显示进度条 |

如果自动安装失败，手动执行：
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple faster-whisper pydub av opencc-python-reimplemented requests tqdm

## 使用步骤

### 第一步：填写术语表
点击 选择视频 后，工具会弹出一个窗口让你填写术语表。

术语表有四个区域：
1. 作品名称
2. 角色名对照表（日文 -> 中文）
3. 特殊术语对照表（日文 -> 中文）
4. 固定短句（日文 -> 中文）

填写格式：
ロイド・フォージャー -> 劳埃德·福杰（黄昏）
おはよう -> 早安

### 第二步：开始处理
点击确定后，工具自动完成语音识别、术语替换、翻译、生成字幕。

### 第三步：找到字幕文件
处理完成后，字幕文件保存在视频同一个文件夹里，文件名是 原视频名_字幕.srt。


## 设置选项说明

点击 设置 按钮可以配置：

### 翻译引擎
- 有道、百度、DeepLX、SiliconFlow 翻译
- 每个都可以单独启用/禁用
- 填写对应的 API 密钥和模型名称

### 语音引擎
- 主引擎：选择 siliconflow_stt（云端）或 whisper（本地）
- SiliconFlow STT：可切换模型
  - `FunAudioLLM/SenseVoiceSmall`（默认）
  - `TeleAI/TeleSpeechASR`（日语识别更准确，推荐）
- Whisper：可切换设备（cuda/cpu）和模型大小

### 备用引擎
- 开启后，主引擎失败会自动切换备用引擎


## ⚠️ 语音识别模型选择建议

本工具默认使用 `FunAudioLLM/SenseVoiceSmall` 模型进行语音识别。如果你遇到以下情况：

- 日语视频被识别成中文字符
- 识别结果与视频内容不符
- 识别准确率低

请在 **设置 -> 语音引擎 -> 云端STT模型** 中更换为 `TeleAI/TeleSpeechASR`。

| 模型名称 | 特点 |
|----------|------|
| `FunAudioLLM/SenseVoiceSmall` | 默认模型，多语言支持 |
| `TeleAI/TeleSpeechASR` | **日语识别准确率更高，推荐使用** |
| `Whisper (本地模式)` | 本地运行，日语识别成熟稳定 |

**切换步骤：**
1. 点击程序主界面的 **设置** 按钮
2. 进入 **语音引擎** 标签页
3. 在 **云端STT模型** 下拉菜单中选择 `TeleAI/TeleSpeechASR`
4. 点击 **保存**，重新处理视频

> 提示：语音识别和翻译是两个独立阶段，识别结果正确翻译才会正确。


## 停止处理

处理过程中随时可以点击 停止 按钮。工具会安全中断当前环节，并生成已处理部分的字幕（文件名会带 部分 标识）。

## 常见问题

### Q：为什么翻译结果全是中文？日文原文识别不正确？

A：这是语音识别阶段的问题，默认的 `SenseVoiceSmall` 模型可能将日语误识别为中文。

**解决方法：**
1. 在设置中将语音识别模型更换为 `TeleAI/TeleSpeechASR`
2. 或者切换到 Whisper 本地识别（需要安装 faster-whisper）
3. 更换后重新处理视频即可

### Q：为什么有些句子没翻译？

A：可能是翻译引擎返回了无效结果。如果所有引擎都失败，会保留日文原文。

### Q：术语替换没生效？

A：检查术语表格式是否正确，必须是 日文 -> 中文。

### Q：报错 Model does not exist？

A：检查 SiliconFlow 设置里的模型名称是否正确。

### Q：提示找不到 ffmpeg？

A：工具处理音频使用 WAV 格式，不需要 ffmpeg。警告可以忽略。

### Q：处理速度很慢？

A：语音识别和翻译都需要联网调用 API，可以只启用一个翻译引擎来提速。

### Q：必须要有 GPU 吗？

A：不需要。使用 SiliconFlow 云端识别时，普通电脑即可运行。

### Q：为什么需要联网？

A：语音识别和翻译都调用云端 API，必须联网。
